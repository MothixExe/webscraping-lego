Projet SAÉ 3.VCOD.01 : Collecte automatisée de données web

Dans le cadre du projet SAÉ 3.VCOD.01, intitulé "Collecte automatisée de données web", j'ai développé un site web sur le thème de Lego en utilisant l'API Brikse, permettant de collecter des informations sur les sets Lego disponibles. Ce site web a été créé en combinant un serveur Flask en Python et du JavaScript pour interagir avec l'API et afficher les données de manière dynamique.

Le serveur Flask joue un rôle crucial en gérant les requêtes et en facilitant la communication avec l'API Brikse, tandis que le JavaScript s'occupe de l'affichage interactif des données sur le front-end, offrant ainsi une interface utilisateur fluide et réactive.

Le site permet aux utilisateurs de rechercher et de filtrer des sets Lego selon différents critères tels que le nom, la date du set, ou encore le thème. Il offre une grande variété de thèmes Lego, parmi lesquels on retrouve des univers comme Star Wars, Harry Potter, Technic, Architecture, City, et bien d'autres. Actuellement, la base de données contient 6 000 sets Lego, offrant ainsi une large sélection d'options pour les passionnés et les collectionneurs de Lego. Cette base de données est mise à jour à chaque fois que l'utilisateur visite un nouveau thème.

L'intégration du JavaScript avec Flask permet d'afficher les informations récupérées en temps réel et de proposer une navigation fluide entre les différentes sections du site. Grâce à cette approche, le projet démontre l'utilisation d'outils modernes pour collecter et afficher des données web de manière interactive, tout en mettant en avant l'importance de l'automatisation de la collecte d'informations dans le développement web.
